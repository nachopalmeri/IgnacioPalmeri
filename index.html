Case study principal

JobBot: evidencia de un producto full-stack

Construido como proyecto full-stack para demostrar backend, producto y despliegue, no como claim inflado de seniority.

Demo Repo README CV
Problema

Automatizar la búsqueda laboral sin perder control sobre autenticación, pagos, webhooks y notificaciones.

Solución

Dashboard en Next.js + API en FastAPI con auth segura, persistencia, webhooks y notificaciones por Telegram.

Arquitectura
Next.js dashboard → FastAPI API → PostgreSQL → Auth cookies → Webhooks → Stripe/MercadoPago → Telegram
Decisiones técnicas

Separar UI y API, validar flujos sensibles del lado servidor y dejar trazabilidad de pagos y notificaciones.

Prueba visible

Demo pública, repo, README y CV vinculan el proyecto con evidencia verificable.

Próximos pasos

Sumar observabilidad, mejorar onboarding y endurecer permisos por rol.
